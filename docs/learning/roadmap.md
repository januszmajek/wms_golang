# Learning Roadmap

This is a flexible sequence of backend vertical slices. It is guidance, not a fixed implementation plan.

## Provisional slice order

1. Create a product.
2. Retrieve and list products.
3. Receive stock.
4. Retrieve stock information.
5. Create an order.
6. Cancel an order.
7. Ship an order.
8. Archive an order.

## First slice focus

The first slice should focus on the smallest useful backend flow for creating a product. Persistence may begin in memory if database persistence is not the learning objective of that slice.

## How to adjust the roadmap

Change the order when product questions, architecture decisions, or learning goals make a different slice more useful. Keep each slice small enough to review and explain.
